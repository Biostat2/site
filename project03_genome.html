Aurélien Ginolhac

Human genes analysis

Parse GENCODE data to summarise some key features of the human genome

Download data, set-up

Go on the latest release of GENCODE .

Download the Comprehensive gene annotation as Content and CHR as Regions. Of note, between GTF and GFF3 formats, pick the GFF3 one.

  • install the bioconductor package rtracklayer to parse GFF files

import GFF

  • import the GFF from GENCODE using the import() function from the package rtracklayer. Assign the result as gencode.

Warning

This steps requires some large memory, such as ~ 4 Gb. So your computer needs at least 8 Gb to perform this project. The object gencode below is 1.09 Gb alone.

Tip

import returns an object of class GRanges. Converting to a tibble is more than encouraged Of note, this operation takes like 2-3 minutes. You may want to save the object while testing your workflow

    • Exon sizes

    Intron sizes

    What can make the protein coding bigger then? Maybe the introns. This computation is challenging since we don’t have an easy intron item in the type column associated with the width column that directly provide the length. The idea is then to work per transcript, and subtract the sum of all exons to the transcript length.

    Your workflow could be:

    1. select relevant columns, that makes the process easier to control
    2. keep per transcript the lines of exons and transcript
    3. compute the sum of exons (could add the transcript length, but we have on transcript entry per transcript_id)
    4. move from long to wide format
    5. compute the difference transcript_length - sum(exon_length)

    • plot the density of the intron sums

    • do the distributions of exon sum appear different for the 4 meta-categories?
